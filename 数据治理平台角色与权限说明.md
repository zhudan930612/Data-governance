# 数据治理平台角色与权限说明

## 1. 文档定位与边界

**文档定位**：统一沉淀数据治理平台的角色清单、角色边界、菜单权限、系统自动行为和配置项归属，避免各 PRD 分散定义。

**边界约束**：
- 本文档定义“谁能做什么、哪些动作由系统自动完成、对应配置归谁管”。
- 本文档采用“默认权限 + 可配置”口径；具体授权可按角色配置，但不改变流程定义。
- 本文档不展开页面字段、交互细节、接口参数和数据库结构。
- 各 PRD 只引用本文档中的角色口径，不重复定义角色边界。

## 2. 角色清单

| 角色 | 阶段 | 主要职责 | 边界说明 |
|---|---|---|---|
| 巡查人员 | 当前阶段 | 手动导入照片、按标注规则执行隐患标注 | 对应当前 `S1` 主来源入口；线下拍照属于业务前置动作，初期兼任标注职责，不承担质检和规则定义 |
| 政府端/其他业务平台等扩展来源相关方 | 后续阶段 | 提供扩展来源样本数据 | 对应后续 `S2` 扩展来源接入；当前不作为默认主来源 |
| 数据管理员 | 当前阶段 | 数据导入、样本池管理、进度监控、导出发起 | 不替代标注、质检和知识库专业判断 |
| 数据标注团队 | 后续阶段 | 执行多人多轮标注、负样本处理 | 随业务发展从巡查人员兼任标注的模式中独立，不定义业务标准 |
| 质检员 | 当前阶段 | 全量质检、驳回回流、挂起发起、抽查执行 | 负责质量判断执行，不负责知识库最终规则裁定 |
| 领域工程师（安全专家） | 当前阶段 | 待确认判定、规则审核、知识库审核、抽查标准把控 | 负责专业规则判断，不承担日常导入和标注执行 |
| AI工程师 | 后续阶段 | 使用高质量数据进行训练与优化 | 当前可作为训练侧协同对象接收导出数据，但不纳入正式平台菜单权限 |

## 3. 角色边界

- `质检员` 与 `领域工程师（安全专家）` 不是两个割裂体系，而是同一专业角色链上的不同职责侧重：
  - `质检员` 负责日常质量裁决和抽查执行。
  - `领域工程师（安全专家）` 负责规则争议、待确认判定和知识库审核。
- 文档中的“平台”不再作为人工角色使用。
- 涉及自动分配、自动入池、超时升级、周期触发、通知回写等动作，统一归为`系统自动行为`。

## 4. 默认菜单权限与可配置边界

> 以下为当前默认口径；具体授权可按角色配置，不影响 `S/G/R/P` 流程定义。

| 角色 | 首期菜单/能力范围 |
|---|---|
| 巡查人员 | 照片导入入口、标注工作台、负样本处理；不进入质检与规则治理菜单 |
| 数据管理员 | 数据接入管理、样本池与样本管理、高质量数据池与导出、首页工作台 |
| 数据标注团队 | 当前不纳入默认菜单体系；后续独立后承接标注工作台、负样本处理、首页工作台 |
| 质检员 | 质检裁决与回流、抽查任务、首页工作台 |
| 领域工程师（安全专家） | 待确认判定、知识库审核、知识库维护、抽查标准确认、首页工作台 |
| 政府端/其他业务平台等扩展来源相关方 | 当前不纳入默认菜单体系；`S2` 启用后按来源接入方案提供数据 |
| AI工程师 | 当前不纳入正式菜单体系；如存在训练侧协同，按平台外导出方式承接 |

## 5. 系统自动行为

| 自动行为 | 说明 | 触发位置 |
|---|---|---|
| 标注轮次分配 | 按已配置的最少轮次和分配规则自动流转样本 | 标注与质检协同 |
| G1 通过后自动入池 | 质检通过样本自动进入高质量数据池 | 数据治理运营 |
| 挂起超时升级 | 挂起样本超时后自动升级到领域工程师（安全专家）；仅当其判断为规则缺失时再转知识库链路 | 标注与质检协同 |
| 抽查周期触发 | 按已配置周期生成周/月抽查任务 | 数据治理运营 |
| 口径回写通知 | 知识库更新后通知相关角色采用新口径 | 知识库与规则治理 |
| 口径版本生效 | 按批次冻结、增量生效规则控制新旧口径适用范围 | 知识库与规则治理 |
| 强制复判触发 | 对被明确标记为强制复判的更新生成后续复查任务，仅作用于历史已通过/已入池数据 | 知识库与规则治理/数据治理运营 |
| 全链路留痕 | 自动记录来源、操作、状态变更、回流链路 | 平台基础能力 |

## 6. 配置项归属

**统一归属**：平台基础能力中的`角色权限与配置治理`。

首期应统一管理的配置项包括：
- 角色与菜单映射
- 标注最少轮次
- 标注分配规则
- 挂起超时升级规则
- 抽查周期规则
- 导出权限控制
- 全链路追溯字段口径
- 口径版本生效规则
- 强制复判标记规则

## 7. 口径发布与强制复判权限

- `领域工程师（安全专家）`：定义本次口径更新的影响范围，并确认强制复判候选集。
- `数据治理运营`：根据受影响规则/隐患类别生成候选集并执行复查任务。
- 本文中的`数据治理运营`在强制复判场景下指任务承接职能，不单独新增人工角色；复查执行与判定仍由`质检员`、`领域工程师（安全专家）`承担。
- 首期不默认全量扫描历史样本，也不以人工上传名单作为主路径；批次、时间范围仅作为缩小范围的附加条件。

| 权限动作 | 角色 |
|---|---|
| 知识库内容维护 | 领域工程师（安全专家） |
| 口径发布 | 领域工程师（安全专家） |
| 强制复判标记 | 领域工程师（安全专家） |
| 挂起超时升级处理 | 领域工程师（安全专家） |
| 复查/抽查任务执行 | 质检员、领域工程师（安全专家） |

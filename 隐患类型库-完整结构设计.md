# 隐患类型库 - 完整结构设计

## 文档说明

**文档定位**: 定义隐患类型库的完整数据结构,指导v0.1到v1.0的演进路径

**版本规划**:
- 初始版本: 5-10个隐患类型,支持技术开发
- 正式版本: 20+个隐患类型,支持试点数据标注
- 扩展版本: 根据试点反馈优化和扩展

---

## 一、应用场景

隐患类型库是数据治理平台的核心数据字典,贯穿数据采集、标注、训练、运营全流程。

### 1.1 数据标注阶段

**标注员使用**:
- 查看隐患类型清单,了解需要识别的隐患种类
- 根据识别要点判定照片中的隐患类型
- 参考案例照片进行对比,确保标注准确
- 遇到易混淆情况时,查看易混淆点说明

**质检员使用**:
- 根据详细定义检查标注是否准确
- 参考法律依据验证隐患类型判定
- 对比案例照片评估标注质量

### 1.2 AI模型训练阶段

**数据准备**:
- 按隐患类型统计样本分布,识别数据不平衡问题
- 根据样本数量规划数据采集优先级
- 评估每个类型的数据充足性

**模型评估**:
- 按隐患类型统计模型性能(精确率、召回率)
- 识别模型表现较差的隐患类型
- 针对性补充困难样本

### 1.3 平台运营阶段

**数据采集管理**:
- 查看各隐患类型的样本数量和标注进度
- 根据缺口制定数据采集计划
- 跟踪数据采集完成情况

**质量监控**:
- 监控各隐患类型的标注质量
- 识别标注一致性较低的类型
- 优化标注规范和培训材料

**业务分析**:
- 统计各隐患类型的发现频率
- 分析高风险隐患的分布特征
- 为监管决策提供数据支持

### 1.4 知识管理阶段

**标准化管理**:
- 统一隐患类型定义,避免理解偏差
- 维护法律法规依据,确保合规性
- 积累典型案例,形成知识库

**持续优化**:
- 根据实际应用反馈调整隐患定义
- 补充新发现的隐患类型
- 优化识别要点和易混淆点说明

## 二、隐患类型完整字段定义

每个隐患类型包含以下完整字段:

| 字段名 | 字段类型 | 技术开发 | 数据标注 | 说明 | 示例 |
|--------|---------|------|------|------|------|
| **基础信息** |
| code | String | 必填 | 必填 | 隐患类型唯一编码,格式:H+3位数字 | H001 |
| name | String | 必填 | 必填 | 隐患类型名称,简短易懂(≤15字) | 配电箱漏电保护缺失 |
| category | String | 必填 | 必填 | 一级分类编码 | C01 |
| sub_category | String | 必填 | 必填 | 二级分类编码 | C01-01 |
| risk_level | String | 必填 | 必填 | 风险等级(A/B/C),A=红色/B=橙色/C=黄色 | B |
| description | String | 必填 | 必填 | 简要描述(1-2句话) | 配电箱漏保缺失或失效,存在触电风险 |
| status | String | 必填 | 必填 | 状态(active/deprecated) | active |
| version | String | 必填 | 必填 | 引入版本 | 0.1 |
| applicable_region | String | 必填 | 必填 | 适用地区（国家/行业/地方） | 上海市 |
| keywords | Array[String] | 必填 | 必填 | 检索关键词 | ["临时用电","漏电保护"] |
| responsibility_owner | String | 选填 | 必填 | 责任主体（班组/分包/项目部等） | 电工班组 |
| location_description | String | 选填 | 必填 | 位置描述（楼层/区域/点位） | 2号楼东侧临时配电区 |
| **详细定义** |
| detailed_definition | String | - | 必填 | 详细定义,包含判定标准 | 配电箱应配置有效漏电保护装置... |
| identification_points | Array[String] | - | 必填 | 识别要点(3-5条) | ["检查钢管表面标识"...] |
| typical_scenarios | Array[String] | - | 必填 | 典型场景描述 | ["新建工程使用劣质钢管"...] |

| **法律依据** |
| legal_references | Array[Object] | - | 必填 | 法律法规依据（条款级） | [{"standard_name":"建筑施工安全检查标准","standard_code":"JGJ59-2011","chapter":"3","clause":"3.3.4","regulation_status":"有效"}] |
| rectification_measures | Array[String] | - | 必填 | 整改措施(3-5条) | ["更换符合标准的钢管"...] |
| rectification_deadline | String | - | 必填 | 整改期限要求 | 立即整改 |
| accident_risk | String | - | 必填 | 可能导致的事故类型 | 坍塌事故 |
| **案例照片** |
| example_photos | Array[Object] | - | 必填 | 典型案例照片(≥3张) | [{"url":"...","description":"..."}] |

## 三、分级分类体系

### 3.1 一级分类(大类)

v1.0首批优先域（按会议结论）:

| 分类编码 | 分类名称 | 说明 | 预计隐患类型数量 |
|---------|---------|------|----------------|
| C01 | 触电类 | 临时用电、漏电保护、违规接线等 | 6-10个 |
| C02 | 高坠类 | 临边/洞口/高处作业防护缺失等 | 6-10个 |
| C03 | 物体打击类 | 坠物防护、吊装警戒、堆放不当等 | 4-8个 |
| C04 | 机械伤害类 | 机械防护缺失、违规操作、带病运行等 | 4-8个 |
| C05 | 动火与消防类 | 动火审批、防火间距、灭火器配置等 | 4-8个 |
| C99 | 其他类（预留） | 新增场景临时归类，评审后归并 | 预留 |

### 3.2 二级分类(子类)

| 一级分类 | 二级分类编码 | 二级分类名称 | 说明 |
|---------|------------|------------|------|
| 触电类 | C01-01 | 配电箱与漏保问题 | 漏保失效、箱体防护缺失、未上锁等 |
| 触电类 | C01-02 | 线路与接线问题 | 破损电缆、私拉乱接、绝缘不足等 |
| 高坠类 | C02-01 | 临边洞口防护问题 | 栏杆缺失、洞口盖板缺失等 |
| 高坠类 | C02-02 | 高处作业防护问题 | 安全带未系、作业平台不规范等 |
| 物体打击类 | C03-01 | 高处坠物防护问题 | 警戒隔离缺失、防护棚不足等 |
| 物体打击类 | C03-02 | 物料堆放与吊装问题 | 堆放失稳、吊装区警示不足等 |
| 机械伤害类 | C04-01 | 机械防护装置问题 | 防护罩缺失、急停失效等 |
| 机械伤害类 | C04-02 | 机械操作与检修问题 | 带病运行、违章检修等 |
| 动火与消防类 | C05-01 | 动火作业管理问题 | 审批缺失、监护不到位、清场不足等 |
| 动火与消防类 | C05-02 | 消防设施配置问题 | 灭火器缺失、过期、摆放不规范等 |

### 3.3 隐患类型编码规则

**编码格式**: H + 3位数字

**编码分配**:
- H001-H099: 触电类
- H100-H199: 高坠类
- H200-H299: 物体打击类
- H300-H399: 机械伤害类
- H400-H499: 动火与消防类
- H900-H999: 其他类（新增提报临时编码）

**预留空间**: 每个二级分类预留20个编码,支持后续扩展

### 3.4 条款级引用规范

**每条隐患必须绑定至少1条有效法规条款，最小必填字段如下**：
- `standard_name`：标准名称
- `standard_code`：标准编号
- `chapter`：章节
- `clause`：条款
- `regulation_status`：法规状态（有效/废止）

**执行规则**：
- `regulation_status=废止` 的条款不得用于新发布条目，仅保留历史追溯
- 同一隐患可绑定多条法规，但必须指定主依据条款

## 五、数据采集场景建议

### 5.1 优先采集原则

从以下三个维度评估优先级：
- **发生频率高**：在施工现场普遍存在,样本获取容易
- **易于拍摄**：隐患外观明显,普通手机可清晰记录
- **采集成本低**：无需特殊设备或专业背景,进场即可拍摄

### 5.2 推荐采集场景（按优先级排序）

#### 第一优先级：触电类隐患

**推荐理由**：事故高发、可见性强、整改价值高。

| 隐患类型 | 典型表现 | 拍摄建议 |
|---------|---------|---------|
| 漏保失效/缺失 | 配电箱内无漏保或漏保失效标识 | 正面+铭牌近景 |
| 线缆破损裸露 | 绝缘层破损、导线外露 | 俯视近景，突出破损点 |

#### 第二优先级：高坠类隐患

**推荐理由**：重大伤亡风险高，需优先覆盖。

| 隐患类型 | 典型表现 | 拍摄建议 |
|---------|---------|---------|
| 临边/洞口防护缺失 | 洞口无盖板、栏杆缺失 | 中景+整体环境图 |
| 高处作业防护不足 | 作业人员未系安全带 | 中景，包含作业点和防护状态 |

#### 第三优先级：物体打击与机械伤害类隐患

**推荐理由**：现场高频，适合通过结构化场景快速积累样本。

| 隐患类型 | 典型表现 | 拍摄建议 |
|---------|---------|---------|
| 高处坠物防护不足 | 警戒区域缺失、防护棚不足 | 远中景结合 |
| 机械防护装置缺失 | 防护罩缺失、急停不可用 | 设备局部特写+设备全景 |

#### 第四优先级：动火与消防类隐患

**推荐理由**：对火灾事故预防直接相关，法规约束明确。

| 隐患类型 | 典型表现 | 拍摄建议 |
|---------|---------|---------|
| 动火审批或监护缺失 | 无动火票、无现场监护 | 作业区全景+审批信息近景 |
| 灭火器配置不合规 | 缺失、过期或摆放不当 | 器材铭牌特写+现场位置图 |

### 5.3 采集注意事项

- **进场安全**：采集人员需佩戴安全帽,遵守工地安全规定
- **样本多样性**：同一隐患类型需覆盖不同工地、不同角度、不同光线条件
- **避免重复**：同一工地同一处隐患只采集一次,优先覆盖更多工地
- **拍摄质量**：确保隐患特征清晰可见,避免模糊、过曝或逆光

## 六、使用指南

### 6.1 技术开发阶段

技术团队根据第一章字段定义和第三章JSON结构示例,实现平台功能。

### 6.2 数据标注阶段

**标注流程**:
1. 查看隐患类型清单和案例照片
2. 根据识别要点判定隐患类型
3. 参考标注示例进行边界框标注
4. 遇到困难样本查看易混淆点说明

**质检流程**:
1. 检查隐患类型是否准确
2. 检查边界框是否符合标注规范
3. 参考案例照片进行对比

### 6.3 字段管理

**字段补充流程**:
1. 技术开发阶段: 只填写"基础信息"字段
2. 领域工程师到位: 补充"详细定义"、"法律依据"、"案例照片"字段
3. 系统自动维护: "系统维护字段"由平台自动更新

## 七、质量保证

### 7.1 隐患类型定义质量标准

- **清晰性**: 3名领域工程师独立理解测试,一致性 ≥ 90%
- **完整性**: 覆盖小散工程常见场景,覆盖率 ≥ 90%
- **准确性**: 法律依据准确,无错误引用
- **可操作性**: 识别要点明确,标注员能够理解和执行

### 7.2 案例照片质量标准

- **典型性**: 每个类型至少3张典型案例
- **多样性**: 覆盖不同角度、光线、距离
- **清晰度**: 照片清晰,隐患特征明显
- **标注准确性**: 边界框IoU > 0.7

## 九、库迭代机制

### 9.1 条目状态流转

| 场景 | 状态流转 | 说明 |
|------|----------|------|
| 新增条目 | `draft -> reviewing -> active` | 现场提报或评审新增，审核通过后发布 |
| 条目修订 | `active -> reviewing -> active` | 修订不删除历史版本，发布后替换当前生效版本 |
| 条目废止 | `active -> deprecated` | 条款废止或场景不再适用，保留历史追溯 |

### 9.2 触发机制

- 法规更新触发：标准版本变更、条款废止、地方补充指引发布
- 业务反馈触发：现场新增隐患提报、质检高频打回类型
- 训练反馈触发：模型持续误判或漏检的高频类型

### 9.3 发布与追溯要求

- 每次变更必须记录：变更人、变更时间、变更原因、影响范围
- 废止条目不得用于新标注，但历史数据保持可追溯
- 季度至少执行1次全库评审并发布差异清单

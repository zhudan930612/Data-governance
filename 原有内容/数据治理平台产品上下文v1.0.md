## 产品定位

**系统名称**：隐患数据治理平台

**核心定位**：构建安责险相关的数据资产治理体系和平台，通过标准化的数据标注、清洗、验证流程，为AI隐患识别模型训练提供高质量数据集，最终实现自动化隐患识别能力。当前阶段以小散工程安全隐患为切入口，验证数据治理体系的有效性，后续逐步扩展到安责险全领域。

**系统关系**：
- **独立系统**：独立部署的数据治理平台，通过接口从多个数据源获取巡查照片数据
- **数据来源优先级（v1.0 会议结论）**：
  - 事故预防服务巡检数据（主来源，优先建设）
  - 政府端"日常巡查"模块（后置接入，按接口进度排期）
  - 外部购买数据（补充来源，后置接入）
- **数据用途**：
  - **AI模型训练**（核心）：训练垂直领域AI隐患识别模型，反哺到日常巡查场景实现自动识别
  - **知识库建设**：构建隐患案例库，用于标注员培训、安全协管员培训、典型案例展示
  - **数据分析洞察**：隐患分布统计、高发类型识别、区域风险热力图、时间趋势分析
  - **监管决策支持**：生成风险评估报告、优化监管资源配置、识别重点检查区域
  - **保险精算应用**：支持风险定价模型、理赔预测、承保决策
  - **行业标准制定**：为隐患类型标准化、风险等级划分提供数据依据
- **质量管理评估**：评估施工质量、第三方服务质量、整改效果验证
- **数据产品化**（长期）：脱敏后的数据集销售、API服务、行业研究报告

**AI职责边界（v1.0）**：
- AI负责识别照片中的隐患类型与风险点
- 法规依据与整改建议由结构化知识库（标准库+隐患库）匹配输出
- 生产报告文本必须基于有效条款，不由生成式AI自由发挥

**目标用户**：
- **巡查人员**：在现场完成拍照与语音备注，不承担正式标注
- **数据管理员**：负责数据导入、任务分配、质量监控、数据导出
- **数据标注团队**：公司内部专业标注人员，负责精细标注和质量检查
- **质检员**：负责标注质量检查和修正
- **AI训练工程师**：使用标注数据训练AI模型

**现场执行约束（v1.0）**：
- 巡检人员多为计件制，现场动作必须极简
- 移动端默认能力：拍照、语音备注、无隐患确认
- 正式标注统一在PC后台完成，减少现场停留时间

**核心场景**：
- **数据采集**：从政府端平台同步巡查照片、从第三方平台接收照片、手动导入外部数据
- **数据标注**：专业标注团队进行精细标注（目标检测级）
- **质量管控**：交叉验证、数据清洗、质量检查
- **数据输出**：导出符合AI训练要求的标注数据集

**关于现场采集与正式标注分工**：
- 现场端（移动端）：巡查人员拍照并补充语音备注，必要时勾选"无隐患"
- 后台端（PC）：标注员执行正式标注（框选、类型、等级），质检员复核
- 初步备注仅作参考输入，不替代正式标注结果
- 移动端辅助框选不作为v1.0默认能力，后续按模型成熟度评估引入

## 商业模式

**商业模式定位**：
- **战略定位**：构建"技术能力"层的核心竞争力（长期竞争力），通过数据资产壁垒建立持续优势
- **核心价值**：通过数据治理构建数据资产，训练垂直领域AI模型，提升事故预防服务质量
- **商业闭环**：数据治理 → AI模型训练 → 自动化隐患识别 → 提升保险端服务质量 → 降低保险赔付风险 → 持续获得保险业务

**在整体商业模式中的位置**：
- **对保险公司**：提升事故预防服务质量，通过AI自动识别隐患，降低人工成本，提高服务覆盖率
- **对政府**：提升监管效率，通过AI辅助巡查人员快速发现隐患
- **对公司**：构建数据壁垒，AI能力是长期竞争力的基础

**投入策略**：
- **数据治理平台**：内部工具，重点投入，确保数据质量
- **AI模型训练**：基于开源预训练模型（如YOLO）进行垂直领域微调，降低研发成本
- **数据积累**：持续积累业务数据，构建时间壁垒和数据资产壁垒，形成长期竞争优势

## 业务背景

**业务痛点**：
1. **人工巡查效率低**：依赖人工现场巡查发现隐患，覆盖率低、效率低、成本高
2. **隐患识别依赖经验**：巡查人员专业水平参差不齐，容易漏检或误判
3. **缺乏AI训练数据**：市面上没有小散工程安全隐患的标注数据集，无法直接使用通用AI模型
4. **数据质量难保证**：现场照片质量参差不齐，缺乏标准化的标注流程

**解决方案**：
- 构建数据治理平台，建立标准化的数据标注、清洗、验证流程
- 采用灵活的标注模式：
  - 政府端数据：专业团队直接精细标注
  - 事故预防服务数据：可选初步标注 + 专业团队精细标注
  - 外部购买数据：根据已有标注质量决定是否需要重新标注或质检
- 基于目标检测技术（YOLO等），训练垂直领域AI模型
- 积累高质量标注数据集，构建数据壁垒

**依赖的业务基础**：
- 政府端"日常巡查"模块：开发中，待上线后将持续产生现场照片数据
- 自建巡查队伍：规划中，计划为保险公司提供事故预防服务
- 隐患问题库：待建立，需定义标准化的隐患类型和描述，为标注提供知识库支撑

## AI应用目标与数据质量标准

> **设计原则**：数据资产具有多种应用价值(知识库建设、数据分析、监管决策等),但当前阶段的平台规划设计核心聚焦在AI隐患识别场景,以此为目标指导数据治理流程设计。其他应用场景作为数据资产的附加价值,不影响当前的核心流程。

### AI隐患识别的应用场景

**场景1：实时巡查辅助**
- 巡查人员用手机拍照 → AI实时识别隐患 → 自动标注隐患类型和位置 → 巡查人员确认或修正 → 生成巡查记录
- **核心价值**：提升巡查效率，降低漏检率，辅助新手巡查人员快速识别隐患

**场景2：历史照片批量分析**
- 上传历史巡查照片 → AI批量识别隐患 → 生成隐患分析报告 → 发现高风险工程
- **核心价值**：快速分析大量历史数据，识别高风险工程和隐患趋势

**场景3：视频监控智能预警（远期规划）**
- 工地监控视频流 → AI实时检测隐患 → 触发预警 → 通知相关人员
- **核心价值**：7×24小时自动监控，及时发现隐患，降低事故风险

**场景4：智能硬件设备辅助识别（远期规划）**
- 巡查人员佩戴AI眼镜/AR设备 → 实时扫描现场环境 → AI即时识别并标注隐患风险点 → 语音/视觉提示巡查人员 → 一键记录隐患信息
- **核心价值**：解放双手，实时提示，提升现场巡查的便捷性和准确性，特别适合高危环境或复杂场景的快速排查

### AI模型的质量要求

要实现上述场景，AI模型需要达到以下性能指标：

> **说明**：以下为初步目标值，基于业务需求和通用目标检测模型的性能范围设定。由于小散工程隐患识别是全新的垂直领域，这些指标需要在实际训练中验证和动态调整。

| 指标 | 初步目标值 | 业务逻辑 | 评测条件 |
|------|-----------|---------|---------|
| **准确率（Precision）** | > 90% | 识别出的隐患必须是真实隐患，避免误报导致用户不信任，影响系统可用性 | 测试集v1.0，置信度阈值0.5，IoU阈值0.5 |
| **召回率（Recall）** | > 85% | 不能漏掉重要隐患，尤其是高风险隐患，漏检可能导致安全事故 | 测试集v1.0，置信度阈值0.5，IoU阈值0.5 |
| **mAP（mean Average Precision）** | > 0.75 | 综合评估模型在所有隐患类型上的性能，平衡准确率和召回率 | 测试集v1.0，mAP@0.5，所有隐患类型均值 |
| **推理速度** | < 1秒/张 | 支持移动端实时识别，提升用户体验，满足现场巡查即时反馈需求 | 主流Android手机（骁龙8系列），输入分辨率640×640，单张推理 |
| **泛化能力** | 场景覆盖率 > 80%（光线/角度/天气/工程类型四个维度各≥2种） | 确保模型在实际应用中的鲁棒性，覆盖多样化的现场环境 | 测试集v1.0中各维度子集的mAP均≥0.65 |

**指标调整机制**：
- 初期训练（数据量 < 10000张）：重点关注模型收敛性，指标可适当放宽
- 中期优化（数据量 10000-30000张）：逐步逼近目标值，识别困难样本
- 长期迭代（数据量 > 30000张）：达到或超过目标值，持续优化长尾类型

### 高质量数据的标准

反推AI模型要求，数据集需要满足以下标准：

**1. 数据量充足**
- 每个隐患类型至少1000张标注样本
- 常见隐患类型（如高处作业无防护、临边防护缺失）需3000+张样本
- 总数据集规模：初期目标10000张，中期目标30000张，长期目标50000+张
- **阶段覆盖策略**：
  - 初期（10000张）：覆盖约10个核心隐患类型，每类均值约1000张
  - 中期（30000张）：扩展到约20个隐患类型，常见类型3000+张，其余类型1000+张
  - 长期（50000+张）：覆盖完整隐患类型库，各类型达到样本下限，长尾类型通过数据增强补充

**2. 数据多样性**
- 覆盖不同光线条件（晴天、阴天、室内、夜间）
- 覆盖不同拍摄角度（正面、侧面、俯视、仰视）
- 覆盖不同工程类型（房建、市政、装修、拆除）
- 覆盖不同拍摄设备（手机、相机、不同分辨率）

**3. 标注准确性**
- 边界框位置准确：IoU（Intersection over Union）> 0.7
- 隐患类型正确：标注准确率 > 95%
- 隐患属性完整：风险等级、整改建议等属性标注完整

**4. 标注完整性**
- 照片中所有隐患都被标注，不遗漏
- 多隐患场景：一张照片中存在多个隐患时，全部标注

**5. 标注一致性**
- 同类隐患的标注标准统一（如"高处作业无防护"的边界框范围一致）
- 不同标注员对同一类隐患的标注方式保持一致

**6. 数据平衡性**
- 各隐患类型样本数量相对均衡，避免长尾问题
- 长尾类型通过数据增强或重点采集补充样本

**7. 困难样本覆盖**
- 边界模糊场景（如隐患与背景难以区分）
- 多隐患重叠场景（如一张照片中有3个以上隐患）
- 小目标场景（隐患占图像面积 < 5%）
- 特殊光线/角度场景（逆光、强光、极端角度）

## 术语表

> 产品中的专业术语定义，避免 AI 混淆相似概念。

| 术语 | 定义 | 易混淆概念 |
|------|------|------------|
| 数据治理 | 通过标准化的流程对原始数据进行标注、清洗、验证，确保数据质量符合AI训练要求的过程 | 与"数据管理"区别：数据治理强调质量提升过程，数据管理强调存储和组织 |
| 初步标注 | 【工作标准待领域工程师到位后制定v1.0】现场巡查人员在上传照片时，对照片进行简单的隐患标注。可能的标注方式：(1) 仅标注隐患类型（图像级标注）；(2) 比较粗的框选 + 隐患类型（简化的目标检测级标注）。具体标注内容、标注方式、框选精度要求等工作标准由领域工程师制定 | 与"精细标注"区别：初步标注精度要求较低，可能只标注类型或粗略框选；精细标注需要精确框选位置和标注详细属性；初步标注作为可选输入，不作为核心流程依赖 |
| 精细标注 | 专业标注团队使用标注工具，在照片中框选隐患区域（矩形框），并标注隐患类型和属性（目标检测级标注） | 与"初步标注"区别：精细标注需要框选位置和标注详细属性 |
| 目标检测 | AI技术的一种，在图像中识别并定位特定目标（如隐患），输出目标的位置（边界框）和类别 | 与"图像分类"区别：图像分类只判断整张图是否有隐患，目标检测还要定位隐患位置 |
| 交叉验证 | 多个标注员对同一张照片进行独立标注，通过对比结果发现标注差异，提升数据质量 | 与"质量检查"区别：交叉验证是多人独立标注后对比，质检是单人审核 |
| 数据清洗 | 识别并处理标注数据中的异常、错误、重复等问题，确保数据集质量 | 与"数据标注"区别：清洗是对已标注数据的质量处理，标注是对原始数据的加工 |
| 标注任务 | 数据管理员分配给标注员的一批待标注照片，包含任务ID、照片数量、截止时间等信息 | 与"标注记录"区别：任务是分配的工作单元，记录是标注的结果 |
| 数据集 | 经过标注、清洗、验证后，符合AI训练要求的照片和标注文件的集合 | 与"数据池"区别：数据集是已完成标注的高质量数据，数据池是待标注的原始数据 |
| YOLO | You Only Look Once，一种流行的目标检测算法，速度快、精度高，适合实时检测场景 | 与"Faster R-CNN"区别：YOLO是单阶段检测，速度更快；Faster R-CNN是两阶段检测，精度更高 |

## 以终为始的业务流程设计

> 基于AI应用目标和数据质量标准，反向推导的数据治理流程设计。

### 设计理念

**1. 数据质量分层管理**
```
原始数据层（多数据源照片）
    ↓ 质量筛选
可用数据层（过滤无效照片）
    ↓ 数据价值评估
高价值数据池（优先标注）
    ↓ 精细标注
精细标注数据层（目标检测级标注）
    ↓ 多层质量管控
高质量数据层（质检通过）
    ↓ 智能数据集构建
训练数据集（按AI训练要求组织）
```

**2. 闭环反馈机制**
```
数据标注 → AI训练 → 模型评估 → 识别困难样本
    ↓                                    ↑
    └──── 补充标注 ← 优化标注规范 ←──────┘
```

**3. 数据价值优先级**

> 注：此优先级策略适用于数据量较大时（> 5000张），初期数据量不大时全部标注

```
高价值数据（优先标注）：
- 高风险工程照片
- 困难样本（多隐患、小目标、边界模糊）
- 稀缺隐患类型（长尾样本）
- AI模型识别错误的样本（反馈回来的困难样本）

低价值数据（延后标注）：
- 重复场景照片
- 常见隐患类型（已有充足样本）
- 质量差无法修复的照片
```

### 阶段0：数据采集与智能筛选

**目标**：从原始照片中筛选出可标注的照片，并在数据量较大时进行价值评估和优先级排序

```
多数据源照片获取
├─ 政府端"日常巡查"模块同步
├─ 第三方服务机构巡检平台接收
└─ 外部数据手动导入
    ↓
自动质量检测
├─ 技术质量：模糊度、曝光、分辨率
├─ 内容质量：是否包含施工场景
└─ 重复检测：相似照片去重
    ↓
数据价值评估（智能打分，分阶段实施）
├─ 工程风险等级（高风险工程优先）
├─ 隐患类型稀缺度（长尾样本优先）
└─ 场景多样性（新场景优先）
    ↓
    注：初步标注信息不参与优先级评分，仅用于标注工具的智能推荐功能
    ↓
进入分层数据池
├─ 高价值池（优先标注）
├─ 中价值池（常规标注）
└─ 低价值池（暂缓标注）
```

**分阶段实施策略**：
- **初期（数据量 < 5000张）**：只进行质量检测，过滤无效照片后全部标注，不做价值评估和优先级排序
- **中期（数据量 5000-20000张）**：引入简单的价值评估（工程风险等级 × 1.5 + 照片质量 × 1.0），开始优先标注高价值照片
- **后期（数据量 > 20000张）**：全面实施数据价值评估（+ 隐患类型稀缺度 × 2.0 + AI困难样本标识 × 3.0），优化标注资源分配

**说明**：初步标注信息不参与优先级评分，仅用于标注工具的智能推荐功能（如推荐隐患类型），避免对特定数据源产生优先级偏向。

**关键创新点**：
- 初期全量标注，快速积累基础数据集
- 中后期引入数据价值评估，优先标注高价值照片，提升标注资源利用效率

### 阶段1：智能任务分配

**目标**：根据标注员能力和照片特征，智能匹配任务

```
数据池照片
    ↓
任务生成策略
├─ 按隐患类型分组（确保标注一致性）
├─ 按难度分级（简单/中等/困难）
└─ 按工程类型分组（利用标注员专业背景）
    ↓
标注员能力画像
├─ 标注速度（张/小时）
├─ 质检通过率（按隐患类型）
├─ 擅长的隐患类型
└─ 擅长的难度等级
    ↓
智能匹配分配
├─ 新手标注员 → 简单任务（单一隐患、清晰照片）
├─ 熟练标注员 → 中等任务（多隐患、常规场景）
└─ 专家标注员 → 困难任务（边界模糊、复杂场景）
    ↓
交叉验证任务自动生成
└─ 按标注员能力分层设置交叉验证比例
```

**关键创新点**：
- 标注员能力画像，实现人岗匹配
- 任务难度分级，提升标注效率和质量
- 自动生成交叉验证任务，无需人工干预

### 阶段2：高效精细标注

**目标**：提升标注效率，降低标注成本

```
标注员接收任务
    ↓
标注工具智能辅助
├─ 智能推荐隐患类型（基于初步标注）
├─ 相似照片参考（显示同类隐患的标注案例）
├─ 标注模板（常见隐患快速标注）
└─ 实时质量提示（框选过大/过小提醒）
    ↓
标注过程监控
├─ 标注时长异常检测（过快可能质量差）
├─ 标注行为分析（频繁撤销可能不确定）
└─ 实时质量评分（基于规则的初步评分）
    ↓
标注完成提交
└─ 自动进入质量管控流程
```

**关键创新点**：
- 智能推荐和参考案例，降低标注难度
- 实时质量提示，减少低质量标注

**AI辅助预标注（分阶段引入）**：
- **第一阶段（0-3个月）**：纯人工标注，积累5000-8000张高质量数据
- **第二阶段（3-6个月）**：小范围试点AI辅助，选择1-2个常见隐患类型
- **第三阶段（6个月后）**：模型性能达标（mAP > 0.7）后全面推广AI辅助

### 阶段3：多层质量管控

**目标**：确保数据质量达到AI训练要求

```
【第一层：自动质量检测】
标注提交后立即触发
├─ 规则检测
│   ├─ 边界框尺寸合理性（不过大/过小）
│   ├─ 边界框位置合理性（不超出图像边界）
│   ├─ 隐患类型合理性（与初步标注对比）
│   └─ 标注完整性（是否遗漏明显隐患）
└─ 结果
    ├─ 通过 → 进入第二层
    ├─ 不通过 → 退回标注员修正
    └─ 不确定 → 标记为重点质检对象

【第二层：交叉验证（分层比例）】
├─ 新手标注员：50%交叉验证（初期）→ 30%（稳定期）
├─ 熟练标注员：30%交叉验证（初期）→ 10%（稳定期）
└─ 专家标注员：10%交叉验证（初期）→ 5%（稳定期）
    ↓
系统自动对比标注结果
├─ 边界框IoU计算（IoU > 0.7视为一致）
├─ 隐患类型一致性（多数投票）
└─ 标注数量一致性（是否遗漏隐患）
    ↓
一致性评分
├─ 高一致性（> 0.9）→ 直接通过
├─ 中一致性（0.7-0.9）→ 质检员复核
└─ 低一致性（< 0.7）→ 专家标注员重新标注

【第三层：人工质检（分层抽样）】
├─ 新手标注员：抽查50%
├─ 熟练标注员：抽查20%
├─ 专家标注员：抽查10%
├─ 自动质量检测"不确定"的：100%
└─ 交叉验证"中一致性"的：100%

【第四层：数据清洗】
批量数据质量优化
├─ 异常检测（统计异常、分布异常、一致性异常）
├─ 数据修正（格式统一、边界框优化、重复数据去除）
└─ 质量报告（各隐患类型样本数量、标注质量分布、困难样本识别）
```

**关键创新点**：
- 四层质量管控，层层把关
- 自动质量检测 + 交叉验证自动化，减少人工质检成本
- 分层交叉验证和抽样质检，针对性管控，成本可控

### 阶段4：智能数据集构建

**目标**：构建符合AI训练要求的高质量数据集

```
高质量数据池
    ↓
数据集构建策略
├─ 数据平衡性分析
│   ├─ 各隐患类型样本数量统计
│   ├─ 识别长尾类型（样本不足）
│   └─ 识别过度采样类型（样本过多）
├─ 困难样本识别
│   ├─ 多隐患重叠场景
│   ├─ 小目标隐患（占图像面积 < 5%）
│   ├─ 边界模糊隐患
│   └─ 特殊光线/角度场景
├─ 场景多样性分析
│   ├─ 工程类型分布
│   ├─ 拍摄角度分布
│   ├─ 光线条件分布
│   └─ 天气条件分布
└─ 数据增强策略
    ├─ 长尾类型：数据增强（旋转、翻转、色彩变换）
    ├─ 困难样本：重点采样
    └─ 过度采样类型：降采样
    ↓
数据集划分
├─ 训练集（70%）：确保各隐患类型均衡，包含足够的困难样本
├─ 验证集（15%）：与训练集分布一致，用于模型调参和早停
└─ 测试集（15%）：独立采样，包含更多困难样本，用于最终模型评估
    ↓
数据集版本管理
├─ 版本号：v1.0, v1.1, v2.0...
├─ 版本说明：数据来源、样本数量、标注质量、与上一版本的差异
└─ 导出格式：YOLO格式（txt）、COCO格式（json）、数据集说明文档
```

**关键创新点**：
- 数据平衡性分析，避免长尾问题
- 困难样本识别和重点采样，提升模型泛化能力
- 数据增强策略，解决样本不足问题

### 阶段5：AI训练与反馈闭环

**目标**：持续优化数据质量，提升AI模型性能

```
数据集导出
    ↓
AI模型训练（内部技术团队）
├─ 基于YOLO预训练模型微调
├─ 训练过程监控（loss曲线、mAP曲线）
└─ 模型评估（准确率、召回率、mAP、推理速度）
    ↓
模型性能分析
├─ 各隐患类型性能分析
│   ├─ 识别准确率高的类型（> 95%）
│   ├─ 识别准确率低的类型（< 80%）
│   └─ 容易混淆的类型对
├─ 困难样本识别
│   ├─ 模型识别错误的样本
│   ├─ 模型置信度低的样本
│   └─ 模型遗漏的隐患
└─ 数据质量问题反馈
    ├─ 标注错误样本
    ├─ 标注不一致样本
    └─ 标注遗漏样本
    ↓
反馈到数据治理平台
├─ 困难样本补充标注
│   ├─ 识别准确率低的类型 → 补充更多样本
│   ├─ 模型识别错误的样本 → 重新质检
│   └─ 容易混淆的类型 → 细化标注规范
├─ 标注规范优化
│   ├─ 更新标注案例库
│   ├─ 细化边界框标注标准
│   └─ 补充困难场景标注指南
└─ 标注员培训
    ├─ 分析常见标注错误
    ├─ 组织标注培训
    └─ 更新标注员能力画像
    ↓
新一轮数据标注
└─ 重点标注困难样本和长尾类型
    ↓
数据集版本迭代（v1.1, v1.2...）
└─ 持续优化，直到模型性能达标
```

**反馈频率策略**：
- **阶段性反馈**：每个数据集版本迭代后反馈（如每增加3000-5000张）
- **关键节点触发**：模型性能不达标时立即反馈（如mAP < 0.6）

**关键创新点**：
- AI训练结果反馈到数据治理平台，形成闭环
- 困难样本识别和补充标注，针对性提升模型性能
- 标注规范持续优化，提升标注一致性

## 关键业务决策

> 记录业务流程设计中的关键决策点和选择理由，指导后续功能设计。

### 决策1：AI辅助预标注策略

**选择方案**：分阶段引入AI辅助

**实施路径**：
- **第一阶段（0-3个月）**：纯人工标注，积累5000-8000张高质量标注数据
- **第二阶段（3-6个月）**：小范围试点AI辅助，选择1-2个常见隐患类型试点AI预标注
- **第三阶段（6个月后）**：模型性能达标（mAP > 0.7）后全面推广AI辅助

**选择理由**：
1. 避免初期模型误导：初期模型质量差，错误的预标注会误导标注员
2. 建立质量基准：第一阶段纯人工标注，建立高质量的标注基准
3. 验证流程可行性：先验证标注规范、质量管控流程是否有效
4. 降低风险：分阶段试点，及时发现AI辅助带来的问题
5. 成本可控：初期数据量不大，纯人工标注成本可控

### 决策2：交叉验证比例策略

**选择方案**：分层交叉验证 + 动态调整机制

**具体策略**：
- **初期（0-3个月）**：新手50%、熟练30%、专家10%
- **中期（3-6个月）**：根据质检通过率动态调整（±10%）
- **后期（6个月后）**：新手30%、熟练10%、专家5%
- **特殊场景**：新隐患类型、困难样本、AI识别错误样本100%交叉验证

**选择理由**：
1. 针对性管控：新手标注员质量不稳定，需要高比例交叉验证
2. 动态优化：根据质检通过率动态调整，避免资源浪费或质量风险
3. 重点保障：新隐患类型、困难样本100%交叉验证，确保关键数据质量
4. 成本可控：整体交叉验证比例约15-20%，相比全量交叉验证成本降低80%

### 决策3：数据价值评估机制

**选择方案**：分阶段引入数据价值评估机制

**实施策略**：
- **初期（数据量 < 5000张）**：不做价值评估，只进行质量检测（过滤模糊、曝光过度等无效照片），通过质量检测的照片全部标注
- **中期（数据量 5000-20000张）**：引入简单规则评估（工程风险等级 × 1.5 + 照片质量 × 1.0），开始优先标注高价值照片
- **后期（数据量 > 20000张）**：全面实施价值评估（+ 隐患类型稀缺度 × 2.0 + AI困难样本标识 × 3.0）

**说明**：初步标注信息不参与优先级评分，仅用于标注工具的智能推荐功能（如推荐隐患类型），避免对特定数据源产生优先级偏向。

**选择理由**：
1. 初期快速积累：数据量不大时全部标注，快速积累基础数据集（5000-8000张），满足AI模型初步训练需求
2. 资源优化：数据量增大后引入价值评估，优先标注高价值照片，提升标注资源利用效率
3. 解决长尾问题：通过稀缺度评估，优先标注长尾类型
4. 针对性提升：AI反馈的困难样本优先标注，针对性提升模型性能
5. 分阶段实施：初期规则简单（甚至不做评估），容易实施；后期逐步引入复杂规则
6. 透明可解释：评估规则清晰，数据管理员可以理解优先级
7. 避免数据偏置：不使用初步标注作为优先级因子，确保各数据源公平对待

### 决策4：反馈闭环频率策略

**选择方案**：阶段性反馈 + 关键节点触发

**具体策略**：
- **阶段性反馈**：每个数据集版本迭代后反馈（如v1.0 → v1.1，每增加3000-5000张）
- **关键节点触发**：模型性能不达标时立即反馈（如mAP < 0.6，某类隐患准确率 < 0.7）
- **反馈内容**：模型性能指标、困难样本清单、数据质量问题、数据缺口分析、优化建议

**选择理由**：
1. 避免频繁打断：每周反馈会频繁打断标注流程，阶段性反馈更合理
2. 数据量充足：每次反馈时数据量增加足够多，模型性能变化明显
3. 及时响应问题：关键节点触发机制，确保模型性能问题能及时发现和解决
4. 成本可控：AI训练成本较高，阶段性反馈可以降低训练频率
5. 闭环完整：每次反馈都包含具体的优化建议，数据治理平台可以针对性补充数据

## 模块清单

> 描述产品各模块的业务定位、场景、角色和主要流程，帮助 AI 理解业务逻辑。

### 数据采集管理
**定位**：从多个数据源获取巡查照片，建立待标注数据池
**核心场景**：数据管理员配置同步规则，系统自动从政府端平台、第三方服务机构巡检平台同步照片，或手动导入外部数据，并进行初步筛选和分类
**使用角色**：数据管理员
**主要流程**：
```
配置同步规则（同步频率、筛选条件）→ 系统自动同步照片/手动导入
→ 自动去重、过滤无效照片 → 按维度分类 → 进入待标注数据池
```
**与其他模块的关系**：
- → 任务管理：数据池中的照片作为任务分配的来源
- ← 政府端"日常巡查"模块：通过接口同步巡查照片和记录
- ← 第三方服务机构巡检平台：通过接口接收巡检照片和记录
- ← 外部数据：手动导入外部购买的图片数据、公开数据集等

### 任务管理
**定位**：数据管理员创建标注任务并分配给标注员，跟踪任务进度
**核心场景**：数据管理员从数据池中选择照片，创建标注任务，设置优先级和截止时间，分配给标注员
**使用角色**：数据管理员、标注员
**主要流程**：
```
数据管理员创建任务 → 设置任务参数（优先级、截止时间）
→ 分配给标注员 → 标注员接收任务 → 开始标注
→ 任务进度跟踪 → 任务完成
```
**与其他模块的关系**：
- ← 数据采集管理：从数据池获取待标注照片
- → 数据标注：任务分配给标注员后进入标注流程
- → 质量管控：任务完成后进入质检流程

### 数据标注
**定位**：对照片进行精细标注（目标检测级），正式标注统一在平台PC后台完成
**核心场景**：
- 巡查人员在现场完成拍照与语音备注
- 标注员在PC端执行正式标注与提交
**使用角色**：标注员（巡查人员仅提供采集输入）
**主要流程**：
```
标注员打开任务 → 查看照片 → 使用矩形框工具框选隐患区域
→ 从标准库/隐患库映射条目选择隐患类型 → 标注隐患属性（风险等级等）
→ 保存标注 → 下一张照片 → 任务完成提交
```
**标注方法**：采用**边界框（Bounding Box）+ 分类属性**方案
- **框选方式**：矩形边界框（不使用多边形或语义分割）
- **标注内容**：① 隐患区域边界框（矩形，紧贴隐患区域）② 隐患类型（从隐患问题库选择）③ 风险等级（高/中/低）
- **选型对比**：

  | 方法 | 适用场景 | 为何不选 |
  |------|---------|---------|
  | 边界框 ✅ | 目标位置识别，形状规则或不规则均可 | — |
  | 多边形 | 需要精确轮廓（如脚手架破损区域走向） | 当前目标是识别隐患是否存在及位置，不需要精确轮廓；标注效率低 3-5 倍；混用会导致训练数据格式不统一 |
  | 像素级分割 | 需要精确测量隐患面积（如裂缝、渗水范围） | 严重超出当前识别目标，标注成本极高 |
  | 关键点 | 人体姿态检测（如判断安全带佩戴） | 当前隐患类型不涉及姿态识别 |

- **选型原则**：标注方法由 AI 模型目标决定，不因场景类型而变。当前阶段目标是"识别隐患是否存在、在哪里"，边界框覆盖 90% 以上的小散工程隐患类型。若未来需要精确测量隐患范围，再针对性引入多边形。

**标注工具要求**：
- 支持矩形框绘制、拖拽、缩放
- 支持快捷键操作（如：D键绘制框、Delete删除框、方向键切换照片）
- 支持隐患类型快速选择（下拉菜单或快捷键）
- 支持标注历史记录和撤销/重做
- 支持照片缩放、平移查看细节
**与其他模块的关系**：
- ← 任务管理：接收标注任务
- → 质量管控：标注完成后进入质检流程
- ← 隐患问题库：标注时从问题库选择隐患类型

### 质量管控
**定位**：通过交叉验证、质量检查、数据清洗确保标注数据质量
**核心场景**：质检员审核标注结果，修正错误标注；系统自动进行交叉验证和数据清洗
**使用角色**：质检员、数据管理员
**主要流程**：
```
【交叉验证】
多个标注员独立标注同一批照片 → 系统自动对比标注结果
→ 识别标注差异（位置差异、类型差异）→ 标记为待复核
→ 质检员人工复核 → 确定正确标注

【质量检查】
质检员审核标注结果 → 检查框选位置是否准确
→ 检查隐患类型是否正确 → 修正错误标注 → 标记为已通过

【数据清洗】
系统自动识别标注异常（框选过大/过小、类型错误等）
→ 标记为待修正 → 质检员人工修正 → 统一标注格式（转换YOLO/COCO格式等）
```
**质量标准**：
- 框选位置准确：边界框应紧贴隐患区域，不过大或过小
- 隐患类型正确：标注的隐患类型应与实际情况一致
- 标注完整性：照片中所有隐患都应被标注，不遗漏
- 标注一致性：同类隐患的标注方式应保持一致
**与其他模块的关系**：
- ← 数据标注：接收标注完成的数据
- → 数据导出：质检通过的数据进入数据集
- → 任务管理：质检不通过的数据退回重新标注

### 数据导出
**定位**：导出符合AI训练要求的标注数据集
**核心场景**：数据管理员选择质检通过的数据，按训练集/验证集/测试集划分，导出为YOLO或COCO格式
**使用角色**：数据管理员、AI训练工程师
**主要流程**：
```
数据管理员选择导出数据范围（时间、隐患类型等）
→ 设置数据集划分比例（70%/15%/15%）
→ 选择导出格式（YOLO/COCO）
→ 系统生成数据集文件（照片+标注文件）
→ 生成路径文件（按训练集/验证集/测试集分类存储）
→ 生成数据集划分文件（实际路径、样本数量、详细分布等）
→ 生成数据集说明文档 → 打包下载
```
**导出格式**：
- **YOLO格式**：每张照片对应一个txt文件，每行一个目标（类别ID 中心x 中心y 宽度 高度）
- **COCO格式**：所有标注信息存储在一个json文件中，包含图像信息、类别信息、标注信息
**与其他模块的关系**：
- ← 质量管控：只导出质检通过的数据
- → AI模型训练（内部技术团队）：导出的数据集用于AI训练

### 统计分析
**定位**：提供数据治理过程的统计分析和可视化，支持管理决策
**核心场景**：数据管理员查看数据采集量、标注进度、质量指标等统计数据
**使用角色**：数据管理员
**主要流程**：
```
系统自动统计各项指标 → 生成统计报表和图表
→ 数据管理员查看分析 → 发现问题 → 优化流程
```
**核心指标**：
- **数据采集指标**：累计采集照片数、日均采集量、按隐患类型分布
- **标注进度指标**：待标注数量、标注中数量、已完成数量、标注效率（张/小时）
- **质量指标**：质检通过率、交叉验证一致性、标注错误率
- **人员效率指标**：各标注员的标注量、标注速度、质检通过率
**与其他模块的关系**：
- ← 所有模块：汇总各模块的业务数据进行统计分析

- **维护机制**：新增提报、修订审核、废止归档，保留全量版本历史

## 设计原则

> 全产品通用的不可违反规则，所有功能设计必须遵守。

- **数据质量优先**：所有功能设计都应以提升数据质量为核心目标，宁可降低效率也要确保质量
- **标注标准化**：建立统一的标注规范和质量标准，所有标注员必须遵守
- **操作留痕**：所有标注、质检、修正操作必须记录操作人、操作时间、操作内容，确保可追溯
- **质量管控闭环**：标注 → 质检 → 修正 → 再质检，形成闭环，确保数据质量
- **高效标注工具**：标注工具必须支持快捷键、批量操作，提升标注效率
- **数据安全**：照片数据涉及现场信息，必须严格控制访问权限，防止数据泄露
- **格式兼容性**：导出的数据集必须符合主流AI训练框架的格式要求（YOLO/COCO）
- **可扩展性**：系统设计应支持未来扩展到语义分割、实例分割等更精细的标注方式
- **数据版本管理**：每次导出的数据集应有版本号，记录数据来源和标注时间，支持追溯
- **反馈机制**：AI训练工程师应能反馈数据质量问题，数据治理平台根据反馈优化标注流程

## 数据安全与合规基础控制项

> 当前为基础框架，明确最小控制集。完整合规控制矩阵（含验证证据、责任归属）需数据安全负责人到位后补充完善。

| 控制项 | 适用范围 | 基本要求 | 验证方向 |
|--------|---------|---------|---------|
| **访问控制（RBAC）** | 所有平台功能 | 按角色分配最小必要权限；数据导出需审批；外部数据需授权准入 | 检查角色权限矩阵是否覆盖所有功能模块 |
| **传输加密** | 所有数据传输（含接口同步、文件上传下载） | 全链路HTTPS/TLS 1.2+；禁止明文传输照片数据 | 抓包验证无明文传输 |
| **存储加密** | 照片文件、标注文件 | 敏感数据存储加密（待数据安全负责人确认加密方案） | 确认存储服务加密配置 |
| **数据脱敏** | 对外展示、测试环境、外部数据购买 | 照片中涉及人脸、车牌等个人信息需脱敏处理后方可用于训练 | 抽检训练数据集中是否含可识别个人信息 |
| **审计日志** | 数据导入、导出、删除、权限变更操作 | 记录操作人、操作时间、操作对象、操作结果；日志保留周期≥180天 | 验证日志完整性，能否追溯关键操作 |
| **数据保留与删除** | 外部购买数据、巡查照片原始数据 | 明确数据保留期限；超期数据按流程删除；外部购买数据须有授权证明和用途边界说明 | 检查数据生命周期管理规则是否落地 |

## 功能规划原则

> 指导功能优先级判断和投入策略的战略原则。

### 优先级判断标准

功能优先级按以下顺序判断：

1. **数据质量** > 标注效率 > 功能丰富度
   - 确保数据质量的功能必须优先实现（质检、交叉验证、数据清洗）
   - 在保证质量的前提下，提升标注效率（快捷键、批量操作）
   - 功能丰富度作为加分项，但不作为优先级主要依据

2. **核心流程功能**（必须实现）
   - 数据采集：从政府端平台同步照片
   - 任务管理：创建任务、分配任务、跟踪进度
   - 数据标注：目标检测级标注工具
   - 质量管控：质检、交叉验证、数据清洗
   - 数据导出：YOLO/COCO格式导出

3. **效率提升功能**（优先实现）
   - 标注工具快捷键
   - 批量操作（批量分配、批量质检）
   - 标注模板（常见隐患快速标注）
   - 智能推荐（基于初步标注推荐隐患类型）

4. **管理分析功能**（适度投入）
   - 统计分析：标注进度、质量指标
   - 人员效率分析
   - 数据集版本管理

5. **高级功能**（后续规划）
   - 语义分割级标注（像素级精确标注）
   - AI辅助标注（预标注 + 人工修正）
   - 标注质量自动评估

### 功能投入策略

- **数据治理平台**：内部工具，重点投入，确保数据质量
- **标注工具**：核心功能，需要高效易用，支持快捷键和批量操作
- **质量管控**：关键环节，必须严格把关，建立多层质检机制
- **统计分析**：辅助功能，提供基础的进度和质量监控即可

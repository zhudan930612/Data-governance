## 2. 业务流程总览

```
【数据来源】
  S1 巡检员（主来源）现场巡查·拍照·语音备注
  S2 政府端 / 外部数据（后置）接入
        |
        v
  S3 照片上传平台
        |
        v
┌─────────────────────────────────────────┐
│ 平台处理（PC端）                          │
│                                         │
│  S4 多人多轮标注                          │
│  有隐患：框选+类型+等级                    │
│  无隐患：负样本确认                        │
│        |                                │
│        v                                │
│  S5 全量质检（质检员裁决）                 │
│    |通过          |不通过                 │
│    v              v                     │
│  S6 高质量数据池/入库  R1 打回修改         │
│    |  |周/月抽查       附错误说明+参考示例 <-┤
│    |  v                ^                │
│    |  S7 定期抽查（独立运营）              │
│    |  |发现问题          |                │
│    |  +----------------+                │
└────|────────────────────────────────────┘
     |
     v
┌─────────────────────────────────────────┐
│ AI训练与部署                              │
│                                         │
│  S8 数据积累达到训练门槛                   │
│        |                                │
│        v                                │
│  S9 AI模型训练                           │
│        |                                │
│        v                                │
│  S10 评估达标？                           │
│    |是            |否                    │
│    v              v                     │
│  S11 部署到巡查业务平台  R2 输出困难样本清单  │
│  +部署到PC标注平台    |（回流到S4标注）       │
│  作为辅助标注建议      |                    │
└───────────────────|─────────────────────┘
                    |
        +-----------+ （回到 S4 多人多轮标注）

【AI辅助标注说明】
  S12 AI模型部署后，同步接入PC端标注平台：
  · S12.1 标注员打开图片时，AI自动给出隐患识别建议（框选位置+类型+等级）
  · S12.2 标注员可采纳（一键确认）、修改（调整框/类型/等级）或忽略（自行标注）
  · S12.3 AI建议仅作参考，最终标注结果以标注员操作为准，仍需经过 S5 全量质检

【知识库回流（并行）】
  P1 照片上传平台发现未覆盖隐患
   -> P2 提报
   -> P3 审核
   -> P4 更新标准库/隐患库
   -> P5 影响 S4/S5 标注与质检口径
```

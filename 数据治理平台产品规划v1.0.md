> **文档定位**：基于《数据治理平台产品上下文.md》的大模块，拆分子功能并明确 v1.0 规划范围与后续演进方向。

## v1.0 定位与边界

**阶段背景**：AI 上线前的数据积累阶段，团队规模小，采用早期极简模式运作。初期标注员可能由巡检员兼任（白天现场拍照上传，回到电脑后登录平台完成正式标注）；中后期随业务规模扩大，两个角色逐步分开，由专职标注员承担平台标注工作。

**v1.0 核心目标**：打通完整数据流——照片手动导入 → 质量过滤 → 精细标注 → 质检审核 → 数据导出，支撑首批 AI 训练数据集构建。

**v1.0 范围约定**：
- 数据来源：**事故预防服务巡检数据优先**（小程序拍照+语音；未就绪时走手动批量导入兜底）
- 政府端与外部数据：**后置接入**，不作为v1.0阻塞项
- 知识底座：**标准库+隐患库共用同一套主数据**，不区分数据来源
- 标注边界：**正式标注仅在PC后台执行**，移动端不做正式标注
- 质量机制：**同图多人多轮标注 + 全量质检**作为v1.0默认方案
- 权限管理：**完整 RBAC**，按角色分配最小必要权限
- 不含：智能任务分配、AI 辅助预标注、数据价值评估

## 模块二：数据采集管理

> 从外部来源接收照片，过滤无效照片，建立待标注数据池。

### 2.1 手动批量导入

| 功能点 | 备注 |
|--------|------|
| 本地文件夹/zip 批量上传 | 支持 jpg/png，单批次上限待定 |
| 来源类型标记 | 导入时选择来源：事故预防服务/外部购买数据 |
| 导入批次记录 | 记录批次号、导入人、时间、数量、成功/失败数 |
| 导入失败原因查看 | 格式不支持、超限等错误提示 |

### 2.2 自动质量过滤

| 功能点 | 技术方案 | 备注 |
|--------|---------|------|
| 模糊照片过滤 | Laplacian 方差法（CPU，1–5ms/张）：灰度转换 → 计算 Laplacian 算子方差，方差 < 100 判定为模糊 | 阈值需用真实巡查照片标定后调整，不固定 |
| 曝光不足/过度过滤 | 像素亮度直方图统计（CPU，2–3ms/张）：暗像素（0–30）占比 > 40% 判定曝光不足；亮像素（220–255）占比 > 50% 判定曝光过度 | 阈值同样需标定 |
| 重复照片检测 | dHash 感知哈希 + 汉明距离（CPU，< 1ms/张）：每张照片生成 64 位哈希值，入库时与已有哈希比对，汉明距离 ≤ 2 视为重复；10 万张哈希库约 800KB | 重复照片标记后归档，不自动删除 |
| 非施工场景过滤 | MobileNetV2 轻量 CNN 分类器（CPU 30–50ms/张）：基于预训练模型微调，训练"施工场景 vs. 非施工场景"二分类；需领域工程师准备 500–1000 张正/负样本 | 不纳入 v1.0；先由标注员在标注时发现并标记，归入无效照片归档区 |
| 无效照片归档与人工复查 | — | 被过滤照片单独归档并保留过滤原因（模糊/曝光/重复），支持人工复核后重新启用 |

### 2.3 事故预防服务对接

> 分阶段对接：先打通拍照/语音上传，再评估移动端辅助能力。

| 功能点 | 备注 |
|--------|------|
| 小程序照片上传接收 | 纳入 v1.0（若小程序已就绪）；若未就绪则按批量导入兜底，不阻塞主流程 |
| 语音备注转文字接收 | 纳入 v1.0；作为后台正式标注参考，不替代正式标注 |
| 巡检员移动端辅助标注（可选） | 不纳入 v1.0 默认能力；v1.x 根据模型与流程稳定度评估引入 |

### 2.4 政府端接口同步

| 功能点 | 备注 |
|--------|------|
| 政府端"日常巡查"模块增量同步 | 不纳入 v1.0；待政府端接口规范完成对接后排期 |
| 全量同步（初始化） | 不纳入 v1.0；待政府端接口规范完成对接后排期 |

## 模块四：数据标注

> 对照片进行精细标注（边界框 + 隐患类型 + 风险等级）。初期由巡检员兼任，中后期由专职标注员承担；无论角色是否合并，标注工作均在 PC 端平台完成。

### 4.1 标注工具核心功能（PC 端）

| 功能点 | 备注 |
|--------|------|
| 移动端正式标注限制 | v1.0 不支持；移动端仅用于拍照和语音备注 |
| 矩形边界框绘制 | 拖拽绘制，紧贴隐患区域 |
| 边界框拖拽调整（移动/缩放） | |
| 多隐患标注 | 一张照片可添加多个边界框 |
| 隐患类型选择 | 从标准库+隐患库映射条目下拉选择 |
| 风险等级标注 | 高/中/低 三级 |
| 无隐患确认（负样本） | 标注"本张无隐患"，不绘制边界框 |
| 照片缩放与平移 | 放大查看细节 |
| 撤销/重做 | |
| 标注提交 | 完成后提交进入质检流程 |
| 操作留痕 | 记录标注人、标注时间、修改记录 |

### 4.2 标注效率工具

| 功能点 | 备注 |
|--------|------|
| 快捷键支持 | 绘制框、切换类型、翻页等常用操作 |
| 巡检员语音备注查看 | 展示语音转文字备注作为参考 |
| 同类隐患参考案例 | 标注时可查看同类隐患的正确标注示例 |

### 4.3 AI 辅助预标注

| 功能点 | 备注 |
|--------|------|
| AI 辅助预标注（小范围试点） | 不纳入当前版本；需先积累 5000-8000 张训练数据，预计 v2.0 第二阶段 |
| AI 辅助预标注（全面推广） | 不纳入当前版本；模型 mAP > 0.7 后推进，预计 v2.0 第三阶段 |

## 模块五：质量管控

> v1.0 采用同图多人多轮标注 + 全量质检机制：先收敛多轮标注结果，再由质检员全量裁决。

### 5.1 人工质检

| 功能点 | 备注 |
|--------|------|
| 质检队列（按时间顺序） | 展示待质检照片，支持按来源和隐患类型筛选 |
| 多人多轮标注结果汇聚 | 同一照片按系统配置完成多轮标注，自动聚合差异点 |
| 全量质检机制 | 对所有样本进行质检确认，不采用抽检 |
| 查看标注结果（边界框 + 类型） | |
| 通过操作 | 一键通过，照片进入高质量数据池 |
| 打回操作 | 选择错误类型（框选不准/类型选错/漏标）+ 填写说明 |
| 附参考示例 | 打回时可附带同类正确标注案例 |
| 参考示例库管理 | 质检员维护同类隐患正确标注案例库 |
| 打回后重新入队 | 标注员修改后重新进入质检队列 |
| 质检操作记录 | 记录质检员、时间、意见 |

### 5.2 自动规则检测

| 功能点 | 备注 |
|--------|------|
| 边界框超出图像边界检测 | 标注提交时自动触发，提示标注员 |
| 边界框过大/过小检测 | 基于面积占比规则 |
| 质检辅助提示（不替代人工） | 自动检测结果仅作提示，最终由质检员决定 |

### 5.3 交叉验证（后续增强）

| 功能点 | 备注 |
|--------|------|
| 交叉验证任务生成 | 不纳入 v1.0；在多人多轮机制稳定后，v1.x引入扩展对比策略 |
| 标注结果自动对比（IoU 计算） | 不纳入 v1.0；待v1.x引入 |
| 分层验证比例配置 | 不纳入 v1.0；待v1.x引入 |

### 5.4 数据清洗

| 功能点 | 备注 |
|--------|------|
| 批量异常标注检测 | 不纳入 v1.0；预计 v1.x |
| 格式统一转换（YOLO/COCO） | 导出时处理 |
| 重复标注去除 | 不纳入 v1.0；预计 v1.x |

## 模块八：标准库 + 隐患库

> 提供结构化知识底座，支撑标注、质检、AI识别结果解释与整改建议输出。

| 功能点 | 备注 |
|--------|------|
| 标准库管理 | 维护标准名称、编号、版本、生效状态（有效/废止）、章/条 |
| 隐患库管理 | 维护隐患类型、等级、描述、整改建议、关键词、位置描述 |
| 条款映射管理 | 隐患条目与标准条款一对多映射；引用粒度到"标准名+编号+章+条" |
| 隐患条目新增提报口 | 现场发现未覆盖项时可提报，审核通过后入库 |
| 版本与状态流转 | 新增、修订、废止全流程留痕，支持历史追溯 |
| 标注侧检索能力 | 标注时按关键词、事故类型、场景快速查询映射条目 |

**说明**：
- v1.0不单列案例库为硬性前置，案例可作为隐患条目附件逐步补齐
- 标准库+隐患库为平台主数据，不依赖政府端内置隐患库

## v1.0 功能优先级汇总

### P0：MVP 核心流程（必须上线）

- 权限与账号管理（完整 RBAC）
- 手动批量导入 + 自动质量过滤
- 事故预防服务拍照/语音接入（小程序未就绪时走手动导入兜底）
- 照片池状态管理 + 标注员自主领取
- 标注工具（矩形框 + 类型选择 + 无隐患确认 + 撤销/重做）
- 质检工作流（多人多轮标注 + 全量质检 + 通过/打回 + 错误说明）
- 数据导出（YOLO/COCO + 说明文档 + 版本记录）
- 基础统计看板（总量概览 + 类型分布）
- 标准库+隐患库（条款映射 + 检索 + 新增提报口）

### P1：效率提升（MVP 后迭代）

- 快捷键、参考案例、语音备注查看
- 自动规则检测（边界框异常提示）
- 参考示例库管理
- 进度趋势、质检通过率、标注员效率
- 数据导出审批（待合规方案确认）
- 交叉验证（多人独立标注对比）

### 后续版本（v1.x / v2.0）

- 任务分配与进度管理（v1.x，团队扩大后）
- 交叉验证（v1.x，多人多轮机制稳定后引入）
- 政府端接口同步（v1.x，接口就绪后）
- 数据价值评估（v1.x，数据量 > 5000 张）
- AI 辅助预标注（v2.0，模型达标后）

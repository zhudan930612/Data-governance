# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 仓库性质

这是一个**纯文档型产品管理仓库**，无代码、无构建系统、无测试框架。所有工作均围绕 Markdown 文档、Mermaid 流程图和 Excel 模板展开。

## 核心文档体系

| 文档 | 定位 | 约束 |
|---|---|---|
| `数据治理平台产品上下文.md` | 业务背景基线文档，所有其他文档的来源依据 | 内容必须可追溯到来源材料，禁止臆想 |
| `数据治理平台-Mermaid.mmd` | 业务流程图的**唯一真源** | 图文必须与总览表保持节点一致 |
| `数据治理平台业务流程总览表.md` | 流程图的文字解释，供产品架构输入使用 | 节点编号/语义必须与 `.mmd` 完全对齐 |
| `数据治理平台产品架构文档.md` | 模块边界、子功能点、状态与 PRD 切分入口 | 不展开页面交互、字段、接口与库表 |
| `知识库体系设计.md` | 知识库结构设计文档 | 从最终输出结果（报告）倒推知识库设计 |
| `产品文档工作流说明.md` | 可复用的产品文档分层工作流说明 | 标准层仅到 PRD，不含设计稿与开发实施说明 |
| `PRD模板.md` | 页面簇/工作流单元级 PRD 模板 | 每篇 PRD 必须内置”必读清单” |
| `数据治理平台角色与权限说明.md` | 统一定义角色清单、边界、菜单权限、系统自动行为 | 各 PRD 只引用本文档角色口径，不重复定义角色边界 |

## 流程编号体系（当前基线）

- **主流程**：S1–S8
- **关口**：G1（全量质检裁决）、G2（周/月抽查判定）
- **回流**：R1（质检驳回→S4）、R2（抽查问题→S4）
- **知识库并行链路**：P1（发现）→ P2（提报）→ P3（审核）→ P4（更新知识库）→ P5（回写口径）

**禁止**：在沟通时使用 Mermaid 节点变量名（如 H、E、G），必须使用业务编号（S1、G1、R1、P1 等）。

## 文件修改规范

- 文件修改：只能用 Edit/Write 工具，禁止 sed/bash 命令
- 文件读取：只能用 Read/Grep/Glob 工具

### 编号重编排三轮替换规则

批量改编号时必须：
1. 原编号 → 临时占位符（如 `_G1_TMP`）
2. 占位符 → 目标编号
3. Grep 验证无残留 `_TMP`

编号变更需同步 4 个文件：`*-Mermaid.mmd`、`*总览表.md`、对应 SKILL.md 文件（如有引用）。

### 连线变更规范

删除或新增 Mermaid 连线时，必须同步检查两端节点标签文案是否仍然准确。

## 文档新增与维护原则

- 上下文文档变更：使用 `update-context-doc` 技能
- 流程图变更：先改 `.mmd`（唯一真源），再同步总览表
- PRD 编写：按页面簇/工作流单元拆分，必须内置”必读清单”
- 功能文档新增：使用 `write-feature-doc` 技能
- 计划文档保存路径：`.claude/plan/`

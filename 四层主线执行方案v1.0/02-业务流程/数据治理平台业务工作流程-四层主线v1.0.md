# 数据治理平台业务工作流程-四层主线v1.0

> 层级：L2 业务流程
> 文档版本：v1.0
> 更新日期：2026-03-20

---

## 1. 文档定位

本文件定义三类流程：用户旅程、使用流程、业务流程关口。  
不定义页面字段与实现交互细节。

## 2. 用户旅程视图

| 旅程阶段 | 用户目标 | 关键输出 |
|----------|----------|----------|
| J1 现场采集 | 快速记录现场隐患信息 | 原始样本入池 |
| J2 平台治理 | 完成标注与质检闭环 | 高质量样本 |
| J3 数据供给 | 构建训练可用数据集 | 版本化数据集 |
| J4 模型反哺 | 用评估结果优化治理 | 回流治理任务 |

## 3. 使用流程接口表

| 步骤 | 输入 | 输出 | 责任角色 | 触发条件 |
|------|------|------|----------|----------|
| U1 数据准入 | 多来源照片/语音/来源信息 | 可治理样本 | 数据工程师 | 新批次进入平台 |
| U2 正式标注 | 待标注样本+知识口径 | 标注结果 | 标注员 | 样本进入待标注状态 |
| U3 质检裁决 | 标注结果 | 通过/打回结论 | 质检员 | 样本进入待质检状态 |
| U4 数据导出 | 质检通过样本 | 训练数据集 | 数据工程师 | 达到导出条件 |
| U5 训练反馈 | 评估结果+困难样本 | 回流任务 | AI工程师/产品经理 | 指标未达标或新场景出现 |

## 4. 业务流程总览

```text
数据接入
 -> 数据准入
 -> 正式标注
 -> 质检裁决
    -> 通过：入高质量样本池
    -> 不通过：打回重标
 -> 数据导出
 -> AI训练评估
    -> 达标：稳定供给
    -> 未达标：困难样本回流治理
```

## 5. 流程关口决策表

| 关口 | 决策问题 | 准入条件 | 失败回流 |
|------|----------|----------|----------|
| G1 准入关口 | 样本能否入治理链路 | 来源合法、信息完整、可读可用 | 回采集端补拍/补录 |
| G2 标注关口 | 标注能否提交 | 口径一致、信息完整、无关键缺失 | 回标注员修正 |
| G3 质检关口 | 样本能否入库 | 质检通过且无关键错误 | 打回重标并附原因 |
| G4 导出关口 | 数据集能否发布 | 样本范围、版本、追溯信息完整 | 回质检/标注补齐 |
| G5 训练关口 | 模型能否进下一阶段 | 当前阶段指标达标 | 回流困难样本治理 |
| G6 变更关口 | 新信息能否生效 | 决策人确认、版本明确、影响层级清晰 | 回产品经理补齐变更单 |

## 6. 更新机制

- 触发机制：事件驱动（会议结论/政策变化/运营反馈/AI反馈）
- 固化顺序：L1上下文 -> L2流程 -> L3架构 -> L4规格卡
- 节奏机制：每周1次跨层一致性复盘

## 完成定义（Ready for Dev）

每轮迭代在以下条件满足后进入开发：
- 页面规格卡完整（目标、入口、状态、交互、字段规则、异常、验收）
- 上下游依赖已确认
- 验收条件可直接转测试用例

## 验收场景（方案级）

1. 新会议结论只影响上下文时，下游不误改。  
2. 新流程分支出现时，规划层能识别受影响模块。  
3. 模块能力变更时，能定位受影响规格卡。  
4. 跨层冲突可输出冲突清单与修复路径。  
5. 规格卡达到 Ready 标准可直接进入开发。

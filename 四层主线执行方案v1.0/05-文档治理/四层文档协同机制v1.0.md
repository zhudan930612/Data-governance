# 四层文档协同机制v1.0

> 目标：保证四层文档协同更新，不出现跨层口径冲突。
> 文档版本：v1.0
> 更新日期：2026-03-20

## 1. 职责边界

| 层级 | 职责 | 禁止项 |
|------|------|--------|
| L1 产品上下文 | 背景/目标/角色/原则/外部约束 | 实现和页面细节 |
| L2 业务流程 | 旅程、流程、关口决策 | 字段级和交互级设计 |
| L3 架构规划 | 模块能力边界与依赖关系 | 页面细节与字段定义 |
| L4 页面规格卡 | 页面目标、状态、规则、验收 | 脱离上层口径独立定义 |

## 2. 更新顺序（强约束）

L1 -> L2 -> L3 -> L4

## 3. 统一输入

所有新增信息先进入“变更信息单”，再触发更新。

## 4. 冲突处理

1. 上下文冲突先回L1统一口径。
2. 流程冲突先回L2明确关口。
3. 架构冲突先回L3调整边界与依赖。
4. 页面冲突先回L4修规格卡并同步影响清单。

## 5. Ready标准

- 规格卡完整且可测
- 上下游依赖明确
- 验收条件可落测试

# 迁移路线图v1.0

> 目标：在不破坏现有体系的前提下，从旧三层习惯迁移到四层主线。
> 文档版本：v1.0
> 更新日期：2026-03-20

## 阶段1：基线冻结（第1周）

- 冻结当前线上文档版本
- 建立变更信息单作为统一入口
- 明确每周复盘节奏与参与角色

## 阶段2：上下文收敛（第1-2周）

- 上下文只保留稳定业务信息
- 将流程和实现细节下沉到L2/L3/L4

## 阶段3：流程升格（第2-3周）

- 完整维护用户旅程、使用流程、业务关口
- 用关口决策表替代分散“注意事项”

## 阶段4：架构重排（第3-4周）

- 统一模块能力清单
- 建立模块依赖矩阵和版本优先级

## 阶段5：规格卡落地（第4-5周）

- 先做“数据采集管理”试点
- 评审通过后每周迁移1-2个模块

## 阶段6：闭环运行（持续）

- 事件驱动更新 + 周复盘
- 持续输出冲突清单、修复动作、版本记录

## 里程碑验收

1. 变更信息单覆盖率达到100%。
2. 每周复盘持续执行且有结论记录。
3. 试点模块规格卡达到Ready并进入开发。
4. 至少2个模块完成四层映射并稳定运行。

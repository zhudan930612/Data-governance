# 数据治理平台产品架构规划-四层主线v1.0

> 层级：L3 产品架构规划
> 文档版本：v1.0
> 更新日期：2026-03-20

---

## 1. 文档定位

本文件定义模块能力边界、模块关系、版本优先级。  
不展开页面字段和交互细节。

## 2. 模块架构总览

```text
权限与账号管理 -> 覆盖全模块
数据采集管理 -> 待标注照片池 -> 数据标注 -> 质量管控 -> 数据导出
标准库+隐患库 -> 支撑标注、质检、导出
统计看板 <- 汇总采集、标注、质检、导出数据
```

## 3. 模块能力清单

| 模块 | 定位 | 上游输入 | 下游输出 | v1.0核心能力 |
|------|------|----------|----------|--------------|
| 权限与账号管理 | 提供访问控制与审计 | 角色体系与合规要求 | 全模块权限能力 | 账号管理、角色权限、审计留痕 |
| 数据采集管理 | 统一接入与准入治理 | 多来源原始数据 | 可治理样本 | 数据接入、准入校验、批次追踪 |
| 待标注照片池 | 承接样本流转 | 采集治理后的样本 | 待标注/待质检样本 | 状态流转、领取机制、操作追踪 |
| 数据标注 | 产出规范标注结果 | 待标注样本+知识口径 | 待质检结果 | PC端正式标注、有/无隐患统一口径 |
| 质量管控 | 保证样本质量一致性 | 标注结果 | 入库样本或打回任务 | 全量质检、打回修正、问题归因 |
| 数据导出 | 形成训练可用数据集 | 质检通过样本 | 版本化训练数据集 | 导出构建、版本记录、可追溯 |
| 统计看板 | 支撑管理与优化 | 全流程运行数据 | 监控与决策依据 | 总览、进度质量、效率分析 |
| 标准库+隐患库 | 提供知识底座 | 领域规则与审核结果 | 统一业务口径 | 条款管理、映射关系、版本治理 |

## 4. 模块依赖矩阵

| 模块 | 关键依赖 | 影响模块 |
|------|----------|----------|
| 数据采集管理 | 权限与账号管理 | 待标注照片池、数据标注 |
| 待标注照片池 | 数据采集管理 | 数据标注、质量管控 |
| 数据标注 | 待标注照片池、标准库+隐患库 | 质量管控 |
| 质量管控 | 数据标注 | 数据导出、统计看板 |
| 数据导出 | 质量管控 | AI训练（外部） |
| 统计看板 | 全流程数据 | 管理决策 |

## 5. 版本优先级

### P0（v1.0必须上线）
- 权限与账号管理基础能力
- 数据采集管理与准入治理
- 待标注照片池状态流转
- 正式标注与全量质检闭环
- 数据导出与版本追溯
- 标准库+隐患库基础治理

### P1（v1.x效率增强）
- 任务分配与优先级策略
- 自动规则辅助与交叉验证
- 导出审批与高级统计分析

### P2（v2.0智能增强）
- AI辅助标注规模化
- 困难样本智能治理与价值评估

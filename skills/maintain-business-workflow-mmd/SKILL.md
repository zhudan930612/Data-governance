# 维护业务流程 `.mmd` 图源

## 技能作用

- 产出并维护唯一流程图真源：`[产品名]-Mermaid.mmd`。
- 固化当前确认版业务流程骨架，避免后续流程口径漂移。
- 让流程图可直接支撑总览说明文档与产品架构拆分。

## 使用场景

- 场景A 首次生成：基于产品上下文产出首版流程图。
- 场景B 增量更新：上下文或业务事实变化后同步更新流程图。
- 场景C 直接调整：用户直接要求改步骤、分支、回流、并行链路。
## 技能关系与调用边界

- 本技能只负责维护图源文件：`[产品名]-Mermaid.mmd`。
- 本技能不负责维护解释文档：`[产品名]业务流程总览表.md`。
- 若需求是“改流程图结构/节点/分支/回流/并行链路”，调用本技能。
- 若需求是“解释流程口径、输出架构输入清单、同步说明文档”，调用 `maintain-business-workflow-doc`。
- 推荐调用顺序固定：先调用本技能更新 `.mmd`，再调用 `maintain-business-workflow-doc` 同步解释文档。

## 0. 产物契约（硬规则）

- 唯一图源文件：`[产品名]-Mermaid.mmd`
- 默认语法：`flowchart TD`
- 默认主题行：`%%{init: {'theme':'mc'}}%%`
- 本技能只维护 `.mmd`；解释文档由 `maintain-business-workflow-doc` 维护。

## 1. 输入边界与来源规则

- 只使用可见来源：`[产品名]产品上下文.md`、会议纪要、用户补充、现有流程文档。
- 首次生成时，上下文文档为首要来源。
- 缺少关键来源时标注“待补充”，不得臆想流程事实。

## 2. 默认流程基线（按当前已确认 `.mmd`）

> 无用户明确要求时，必须遵循以下骨架与链路。

### 2.1 固定子图分区

- `SRC`：数据来源
- `PLT`：平台处理（PC端）
- `AIT`：AI训练与部署
- `LOOP`：知识库回流

### 2.2 固定主链与关口

- 来源汇聚：`A1 -> B`、`A2 -> B`
- 平台主链：`B -> E -> G`
- G2 判定：
  - 通过：`G -> H`
  - 不通过：`G -> R2 -> E`
- 抽查链路（G3）：
  - `H -.-> G3`
  - 是：`G3 -> R3 -> G`
  - 否：`G3 -> G4`
- 训练启动（G4）：
  - 否：`G4 -> ACC -> H`
  - 是：`G4 -> M -> F`
- 反哺：`F -.-> E`、`F -.-> G`

### 2.3 固定并行回写链路（P/KG4）

- `B -> U -> V -> W -> X -> Y`
- `Y` 必须回写：`E`、`G`、`G3`
- 复判分支：`Y -> KG4`
  - 是：`KG4 -> G4`
  - 否：`KG4 -> Z`

### 2.4 当前节点语义（默认）

- `E`：多人多轮标注
- `G`：全量质检（质检员裁决）
- `H`：高质量数据池/入库
- `M`：S8 AI训练与部署模块（当前阶段模块级表达）
- `Y`：P5 回写口径（标注/质检/抽查）

## 3. 可调整边界（需用户明确指令）

- 可改：节点文案、子图标题、线条样式、布局细节。
- 慎改：G2/G3/G4 与 R2/R3/KG4 的判定与去向。
- 禁改（默认）：删除关键关口、删除回流闭环、删除并行回写锚点。

## 4. 命名与标签规则

- 节点命名优先短句，可理解、可追溯。
- 判定节点用问句或条件句（如 `G3 抽查发现问题？`）。
- 回流节点必须保留 `R` 编号（如 `R2`、`R3`）。
- 并行回写节点必须保留 `P` 编号（至少 `P1~P5` 可追踪）。

## 5. 一致性自检（必做）

- S/G/R/P 与 KG4 链路完整且闭环。
- `G2/G3/G4` 每个关口均有明确去向。
- 不存在死链节点、悬空回流或无归属并行链路。
- 与产品上下文核心口径一致。
- 主链完整性优先于角色展示，不得因角色展示破坏 G/R/P 闭环。
- 无无来源新增事实、无幽灵项。

## 6. 输出闭环与更新记录联动

- 输出当前状态：`维护中` 或 `已冻结`。
- `维护中`：说明缺口与待确认项。
- `已冻结`：说明可进入产品架构文档。
- 默认不改总览文档更新记录。
- 仅当用户明确提出“更新记录/写入更新记录”时，输出 `.mmd` 变更摘要供总览文档写入。

## 7. 测试场景

- 首次生成：基于上下文可生成完整骨架（SRC/PLT/AIT/LOOP + G2/G3/G4 + R2/R3 + KG4）。
- 增量更新：改动后链路仍闭环、编号一致、无死链。
- 分支完整性：G2/G3/G4/KG4 均可定位双向去向。
- 回写完整性：`Y` 同时回写到 `E/G/G3`。

## 禁止项

- 禁止把推断写成事实。
- 禁止写入页面字段、接口参数、阈值数值、数据库结构。
- 禁止删减关键关口后不补替代链路。
- 禁止仅输出抽象口号而无可执行流程节点。
